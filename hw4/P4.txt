1.Doubles and floats. 
===============================================================================
My HW4 starts off with a direct continuation of my j-- in HW3 as a starting 
point. 
In reserve words TOKEN, I added DOUBLE to check for double token in the 
reserve word. 
In reserve words TOKEN, I added FLOAT to check for float token in the 
reserve word. 
In reserve words TOKEN, I added LONG to check for long token in the 
reserve word. 

In operators TOKEN, I added COLON to check for colon token in the 
reserve word. 

In identifiers TOKEN, I added #EXPONENT to check either 'E' or 'e' for 
exponents in the literal. I also added #SIGN to check either '+' or '-' for 
either magnitude of the values in exponent in the literal. I added #SUFFIX 
to check either 'F' or 'f' for in the literal in order to be able to check 
whether the value is a float, and check either 'D' or 'd' for in the literal 
in order to be able to check whether the value is a double. 

In literal TOKEN, I added DOUBLE_LITERAL to check for colon token in the 
reserve word.
For the first sign of a double literal indicates if there is a dot after the 
zero digit followed by the next digit. The number is multiplied by either a 
non zero digit exponent. The exponent is checked whether it is a signed non 
zero digit, followed by whether it's digit is multiplied by a suffix in 
order to indicate the number/value is a float or double. 
For the second sign of a double literal indicates if there is a digit 
followed by a dot after the digit followed by the next digit. The number is 
multiplied by either a non zero digit exponent. The exponent is checked 
whether it is a signed non zero digit, followed by whether it's digit is 
multiplied by a suffix in order to indicate the number/value is a float or 
double.
For the third sign of a double literal indicates if there is a digit that is 
multiplied by either a non zero digit exponent. The exponent is checked 
whether it's signed non zero digit is multiplied by a suffix in order to 
indicate the number/value is a float or double. 
For the fourth sign of a double literal indicates if there is a digit that is 
multiplied by either a non zero digit exponent. The exponent is checked 
whether it's suffix is multiplied by a signed non zero digit in order to 
indicate the number/value is a float or double. 

Both FLOAT_LITERAL, and LONG_LITERAL would be duplicates of DOUBLE_LITERAL, 
since it they are all technically floats but vary in level of precision. 

In private Type basicType(), I modified the try statement to by adding a 
DOUBLE instance to parse the doubles in a basic type.

In private JExpression literal(), I modified the try statement by adding a 
DOUBLE_LITERAL expression to implement doubles in the literal similiar to 
the expressions within the try statement. The same was done with LONG_LITERAL,
and FLOAT_LITERAL as with done with DOUBLE_LITERAL.

2.The classic and enhanced for-statements.
===============================================================================

In reserve words TOKEN, I added FOR to check for the for token in the 
reserve words.
 
 In private JStatement statement(), I added both JExpression 'array' to 
provide an array for the for statements  , and 'update' to be able to 
update the for statements. I also added JStatement 'initialize' to 
initialize the for statements, and 'terminate' to terminate the for 
statements. I also added a JBlock 'bodyblock' to indicate a body for the 
for statements. 
	Within the try statement of the statement(), I added a <FOR> {...} in 
order for j--.jj to parse a for statement. The FOR statement begins with 
a token in the line with a left parentheses or '(' followed with a 
variable declarator instantiated. Within the brackets, the parser looks 
ahead of the string to check for an array, checks if expresion is 
terminated, and updates the expression. It is finally closed by a right 
parantheses or ')'. A bodyblock is instantiated to contain the body in 
the for statement. If the for conditions are not terminated then it 
returns a for statement, otherwise an enhanced for statement is returned 
to indicate a for statement with multiple conditions divided by 
semicolons in a for loop. 

===============================================================================
3.The interface declaration and the implements clause in the class 
declaration.
===============================================================================

In reserve words TOKEN, I added IMPLEMENTS and INTERFACE to check for 
implements and interface tokens in the reserve words. 

Within private classDeclaration(), in the try statement, I included 
IMPLEMENTS to instantiated an qualified identifier to be able to 
implement an qualified identifier.

In j--.jj I added a private JInterfaceDeclaration known as 
interfaceDeclaration() that is meant to parse an interface declation, it is 
very much based on the JClassDeclaration classDeclaration(); except that a 
interfaceBody instead of classBody that is instantiated. Within the try 
statement, a <INTERFACE> is in place instead of <CLASS>. Much like 
classDeclaration(), the superclass implements and extends the qualified 
identifier. Instead of a classBody() instantiated, an interfaceBody() is 
instantiated. Finally a JInterfaceDeclaration() is returned with an 
interfaceBody instead of a JClassDeclaration(). 

In j--.jj I added a private ArrayList<JMember> known as interfaceBody(), 
that is exaclty based of off of the ArrayList classBody(), this will allow 
interface declaration to parse a interface body.
 
Within private JAST typeDeclaration(), in the try statement, I included a 
typeDecl that can be either  to instantiated an interface declaration or 
class declarationto be able to parse a type declaration.
